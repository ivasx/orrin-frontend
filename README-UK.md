# Orrin Frontend

## Огляд проєкту

Orrin — це веб-застосунок корпоративного класу, що поєднує високоякісний стримінг музики з розширеними можливостями соціальних мереж. Цей репозиторій містить високооптимізовану фронтенд-архітектуру, розроблену для забезпечення плавного та продуктивного користувацького досвіду. Виходячи за межі стандартного мінімально життєздатного продукту, фронтенд Orrin використовує сучасні вебтехнології для управління складним станом застосунку, безпечними потоками автентифікації та відтворенням медіа, чутливим до затримок.

## Основна архітектура та технічні особливості

Фронтенд Orrin зазнав значного рефакторингу для впровадження надійних архітектурних патернів, придатних для масштабування та підтримки.

### Високопродуктивний аудіодвижок

Щоб уникнути блокування потоку UI під час безперервного відтворення аудіо, застосунок використовує архітектуру Pub/Sub без повторного рендерингу. UI-компоненти медіаплеєра, а саме елементи керування часом та індикатори прогресу, використовують `requestAnimationFrame` для відстеження з частотою 60FPS. Цей підхід повністю обходить стандартний цикл рендерингу React для високочастотних оновлень, забезпечуючи плавні анімації та безперебійне відтворення аудіо незалежно від одночасних мутацій DOM.

### Корпоративний мережевий рівень

Безпека та безперервність сесій управляються через власний надійний мережевий рівень. Автентифікація покладається на зберігання JSON Web Token (JWT) в пам'яті для зменшення векторів атак Cross-Site Scripting (XSS). Застосунок використовує вдосконалений перехоплювач fetch із чергою невдалих запитів. При отриманні відповіді `401 Unauthorized` перехоплювач призупиняє вихідні запити, у фоновому режимі тихо оновлює токен і потім скидає та повторює запити з черги без порушення роботи користувача.

### Управління станом та оптимістичний UI

Глобальний стан сервера, кешування та синхронізація даних організовані за допомогою TanStack React Query v5. Це забезпечує вбудовані механізми фонового повторного отримання даних та інвалідації застарілих даних. Для важких навантажень на дані, таких як стрічки активності та дискографії, застосунок реалізує нескінченне прокручування через Intersection Observer API. Для забезпечення сприйманих взаємодій з нульовою затримкою для соціальних функцій (наприклад, лайки, репости та коментарі) активно використовуються оновлення оптимістичного UI. Ці оновлення миттєво мутують локальний кеш, поки мережевий запит очікує, разом з автоматичними механізмами відкату у разі невдачі мутації.

### Розширений RBAC та маршрутизація

Маршрутизація застосунку глибоко інтегрована з суворою системою контролю доступу на основі ресурсів (RBAC). Використовуючи React Router, архітектура забезпечує суворий захист маршрутів з перевіркою дозволів перед рендерингом. Система точно розрізняє різні ролі користувачів, зокрема обмежуючи функції та доступ до даних для неавтентифікованих гостей, стандартних користувачів та менеджерів виконавців. Ця перевірка на основі ресурсів гарантує, що користувачі взаємодіють лише з авторизованими кінцевими точками та UI-представленнями.

## Структура проєкту

Кодова база організована у модульну, орієнтовану на функціональність структуру директорій для підтримки корпоративної масштабованості.

```text
orrin-frontend/
├── public/                 # Статичні ресурси та медіафайли
├── src/
│   ├── assets/             # Зібрані статичні ресурси (зображення, вектори)
│   ├── components/         # Повторно використовувані, доменно-незалежні UI-компоненти
│   ├── constants/          # Загальні константи та значення конфігурації
│   ├── context/            # Провайдери React Context для глобального стану клієнта (Аудіо, Налаштування)
│   ├── data/               # Моковані дані для офлайн-розробки
│   ├── hooks/              # Кастомні React-хуки (обробка аудіо, логіка макету, спостерігачі)
│   ├── i18n/               # Словники локалізації та конфігурація
│   ├── layouts/            # Структурні обгортки сторінок (MainLayout, HeaderOnlyLayout)
│   ├── pages/              # Компоненти сторінок для конкретних представлень
│   ├── routes/             # Визначення маршрутів та логіка захисту RBAC
│   ├── services/           # Мережевий рівень, API-клієнти та конфігурації перехоплювачів
│   ├── utils/              # Чисті утилітарні функції та форматери
│   ├── App.jsx             # Кореневий компонент застосунку
│   └── main.jsx            # Точка входу застосунку та ін'єкція провайдерів
├── .env.dist               # Шаблон змінних середовища
├── Dockerfile              # Специфікація контейнеризації для продакшену
├── package.json            # Маніфест залежностей та скрипти
└── vite.config.js          # Конфігурація збирання та розробки Vite
```

## Початок роботи

Дотримуйтесь цих інструкцій для налаштування середовища розробки Orrin Frontend.

### Передумови

Переконайтеся, що ваше локальне середовище розробки відповідає таким вимогам:

- Node.js (версія 20.x або вище)
- npm (версія 8.x або вище)

### Встановлення

Клонуйте репозиторій на локальну машину:

```bash
git clone https://github.com/ivasx/orrin-frontend.git
cd orrin-frontend
```

Встановіть необхідні залежності:

```bash
npm install
```

### Налаштування змінних середовища

Застосунку потрібні певні змінні середовища для коректної роботи.

Створіть файл `.env` у кореневій директорії. Ви можете використати `.env.dist` як шаблон.

Налаштуйте необхідні змінні:

```env
# Вкажіть цільовий API-бекенд.
VITE_API_BASE_URL=http://127.0.0.1:8000

# Встановіть 'false' для зв'язку з реальним бекендом. Встановіть 'true' для використання локальних мокових даних.
VITE_USE_MOCK_DATA=false
```

### Запуск локально

Для запуску сервера розробки за допомогою Vite:

```bash
npm run dev
```

Застосунок буде доступний за адресою `http://localhost:5173`.

### Продакшн-збирання

Для компіляції застосунку для продакшн-розгортання:

```bash
npm run build
```

Для попереднього перегляду скомпільованого продакшн-збирання локально:

```bash
npm run preview
```
